## 使用说明

1. **版本号规范**：遵循语义化版本（SemVer）
   - `MAJOR` - 不兼容的重大变更（如 API 破坏性变更）
   - `MINOR` - 向下兼容的功能新增（如新增 API 端点）
   - `PATCH` - 向下兼容的问题修复

2. **变更类型**：
   - `功能发布` - 新版本正式发布
   - `功能更新` - 功能迭代优化
   - `Bug修复` - 问题修复
   - `重要更新` - 重大架构/技术变更

3. **维护建议**：
   - 新版本发布时，在「版本历史」添加条目
   - 涉及 API 变更时，同步更新「API 变更记录」
   - 涉及数据库变更时，同步更新「数据库变更记录」
   - 保持简洁，无需记录过于细节的内容
